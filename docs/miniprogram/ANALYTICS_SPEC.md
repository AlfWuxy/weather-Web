# 微信小程序首版指标与隐私边界

## 目标

首版只回答三个产品问题：用户能否完成一次有效家庭照护启用、启用后第二周是否继续行动、家庭分享来源账号的登录后有效启用表现如何。公开天气浏览使用微信公众平台自带的聚合统计，不在客户端安装第三方统计 SDK。

## 核心指标

### 7 日有效照护启用率

- 分母：在报表统计窗口内首次完成微信登录，并在当地日期 D7 开始时已获得完整 D0 至 D6 观察期的去重账号数。
- 分子：首次登录当地日期 D0 至 D6 内，同时创建至少一份家人档案，并提交至少一次 `actions_done_count >= 1` 的有效行动确认账号数。
- 排除：零行动提交、测试账号、后台脚本和重复重试。
- 漏斗辅助指标：分别报告“登录到建档”和“建档到有效行动”的转化，不能用已建档用户替代总分母。

### 第二周行动留存率

- 分母：已完成 7 日有效照护启用，并在首次有效行动当地日期 D15 开始时拥有完整 D8 至 D14 观察期的去重账号数。
- 分子：首次有效行动后的当地日期 D8 至 D14 内，再次提交至少一项行动的账号数。
- 统计粒度：按首次有效行动所在本地自然周形成 cohort，展示有效启用人数、D15 成熟分母、第二周留存人数和比例，不展示单个账号轨迹。

### 家庭分享来源登录后有效启用率与直接访问对照

- 口径：只有“分享给家人”明确按钮生成的白名单路径携带 `from=family_share`。普通右上角分享和朋友圈分享不携带家庭来源。
- 家庭分享来源登录后有效启用率：家庭分享来源中完成有效启用的账号数，除以该来源已登录且获得完整 D0 至 D6 观察期的账号数。
- 该指标从首次登录开始计算，不代表分享卡打开率、分享落地人数或落地到登录转化率。分享与公开访问的前序触达只使用微信公众平台提供的聚合统计。
- 直接访问来源登录后有效启用率使用相同成熟队列口径，两个来源同时展示人数与比例，避免只比较绝对数量。
- 来源只允许 `direct` 与 `family_share` 两种汇总口径。客户端不上传分享人、接收人、群聊、家人档案或设备信息。
- 前序隐私迁移已清除来源证据的历史账号标为内部 `unknown`，继续计入整体产品指标，但从直接访问与家庭分享的来源对比中排除。

## 首发判读时间表

| 时间点 | 已成熟的证据 | 可以回答的问题 | 仍不能下的结论 |
| --- | --- | --- | --- |
| 发布前 | 埋点、匿名化、测试账号排除和注销清理回归 | 数据链路是否按口径工作 | 任何真实用户增长或留存效果 |
| 当地 D7 | 首批完整 D0 至 D6 队列 | 有效照护启用率，以及 `direct` 与 `family_share` 的同口径差异 | 第二周留存和长期使用价值 |
| 当地 D15 | 首批完整 D8 至 D14 队列 | 第二周行动留存，以及成熟样本中的主要流失节点 | 更长期留存和因果效果 |
| 持续运营 | 微信公众平台聚合触达、自有匿名转化和运行护栏 | 触达问题与产品内转化问题分别出现在哪里 | 单个用户画像、跨应用追踪或缺少成熟分母的来源优劣 |

发布前不生成效果数值，不用 fixture 或测试账号替代上线结论。D7 与 D15 复盘同时报告成熟分母、分子、比例、`unknown` 占比和排除测试账号后的结果；样本不足时保留内部观察，不发布外部结论。

## 事件边界

- 服务器内部事件用于账号登录、档案创建、行动确认和必要的运行结果。客户端不能伪造这些事件。
- 小程序客户端 `/mp/api/v1/events` 当前只允许 `template_copy`，因为这是现有界面唯一需要主动上报的交互。Web `/api/events` 另允许现有反馈入口使用 `feedback_submitted`。
- `meta` 只接受后端白名单中的布尔、整数和枚举。自由文本、列表对象、个人位置和健康内容一律拒绝或丢弃。
- 档案字段名列表 `fields`、`updated_fields` 也不保存，避免从字段组合推断账号正在维护的健康信息。
- `UsageEvent` 不保存 `member_id` 或家庭配对标识。登录后事件只通过内部账号 ID 在 30 天窗口内形成账号级聚合，不保存明文 OpenID。
- 微信身份表将首次登录时间和固定来源枚举保存至账号注销，避免事件清理后把老用户误判为新用户。独立的每日清理 timer 删除 30 天前账号级行为事件。
- 小程序漏斗、试点管理看板和聚合 CSV 最多回看 30 天，不把短期事件数据包装成 90 天口径。
- 账号注销会删除该账号和家庭配对关联的产品事件。
- 内部测试账号通过仅存在于服务器环境的 `ANALYTICS_TEST_USER_IDS` 排除，支持逗号或空格分隔。该配置不进入小程序包，正式体验前应记录测试账号的内部数字 ID。
- 管理看板明确显示测试账号排除是否启用、配置 ID 数和窗口内实际排除数；来源表同时报告 `direct`、`family_share`、`unknown` 的 D7 与 D15 成熟分母。
- CSV 只按本地日期、事件类型和来源导出聚合计数，并应用同一测试账号排除配置；不导出逐事件时间、账号 ID 或 `meta_json`。
- 社区覆盖卡片只使用社区编码，排除测试账号，并仅显示达到 `ANALYTICS_MIN_LOCATION_COUNT` 的聚合行；生产配置强制最少为 3 个家庭，禁止展示自由输入地点。

## 体验与成本护栏

- 公共启动成功率、stale 可读率、接口 P95 延迟和 5xx 比例。
- 30 分钟 timer 成功率、QWeather 月度预算余量和 fail-closed 状态。
- 小程序主包大小、首屏可交互时间、GIS 下载失败率和 GIS 列表模式可用率。
- 产品事件个人敏感字段命中数必须为 0，账号注销后的关联事件残留数必须为 0。

## 发布验收

1. 用 fixture 创建一次直接登录和一次 `family_share` 登录，确认服务端只保存固定来源。
2. 提交零行动，确认不会计入有效启用；再提交一项行动，确认只计一次。
3. 提交自由事件名、数组元数据、姓名、地址和超长元数据，确认接口拒绝或彻底丢弃。
4. 将测试事件时间推进到 31 天前，运行清理并确认删除。
5. 注销测试账号，确认其用户和配对关联事件为 0。
6. 停止天气缓存 timer，单独运行产品事件清理任务，确认仍能删除过期事件且天气服务调用数为 0。
7. 把测试账号 ID 加入 `ANALYTICS_TEST_USER_IDS`，确认登录、启用、留存和家庭分享聚合均不包含该账号。

正式运营报表只展示聚合人数、比例和运行护栏。样本很小时不对外展示细分结果，避免间接识别个人。
