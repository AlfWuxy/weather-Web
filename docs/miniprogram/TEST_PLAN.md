# 小程序多轮验收计划

所有常规轮次禁用真实 QWeather key，使用固定 fixture 和数据库快照。

## 第 1 轮：静态与结构

- JS 语法、JSON 解析、页面四件套、app.json 路径、sitemap。
- 扫描 AppSecret、天气 key、HTTP 地址、占位域名和 QWeather host。
- 检查主包体积与 GIS 数据未进入代码包。

## 第 2 轮：API 契约

- bootstrap、public community、微信登录、不同 OpenID 账号隔离与隐私版本轮换。
- 老人、评估、日记、用药、行动接口的成功与错误响应。
- owner scope、限流、输入长度、非法 JSON、401 与撤销会话。
- 断言普通接口不含原始病历敏感字段。

## 第 3 轮：缓存与额度

- 29:59 命中，30:01 stale。
- 多页面并发只发一个 bootstrap。
- 多老人共享一个 snapshot_id。
- 月预算为 0、Redis 不可用和上游不可用时 QWeather 请求数为 0。
- bootstrap 只读缓存，不允许临时刷新。

## 第 4 轮：核心用户旅程

- 首次打开无需登录即可完成天气、预报、预警、行动、社区和避暑查看。
- “添加家人”在首次 `onLoad/onShow` 并发、隐藏返回和授权状态网络失败后都能退出加载态并安全重试。
- 避暑资源定位每次先显示用途确认；允许后只调用一次 GCJ-02 定位并端内排序，拒绝或失败后可手选社区，离页后清除用户坐标。
- 用户同意隐私后微信登录。
- 新增老人，完成健康筛查、行动话术、行动确认、日记和用药记录。
- 退出后公共功能继续可用，私有页面要求重新登录。

## 第 5 轮：异常与可访问性

- 无网、慢网、5xx、401、空数据、旧快照、GIS 失败。
- 大字号、长文本、窄屏、深色系统设置和点击区域。
- 老人可读文案、色彩对比、风险颜色之外的文字标签。
- 严重症状和求助路径清晰。
- 用药和求助明确为记录能力，不出现自动送达承诺。

## 第 6 轮：安全与隐私

- Token、OpenID hash、日志和错误响应。
- 分享、剪贴板、单次定位、地图、推送和本地 storage 的最小数据范围；确认用户坐标不进入网络请求、日志、分析、分享或持久存储。
- 账号解绑或删除流程。
- 服务端 AppSecret 与天气 key 不进入响应和包体。

## 第 7 轮：开发者工具与真机

- 微信开发者工具编译、预览、网络面板、合法域名。
- Android 与 iOS 真机走查。
- 体验版由非开发者完成一次端到端操作。
- 记录截图、版本号、commit、测试结果和已知限制。

## 最终受控真实联调

前 7 轮全部通过后再执行。先读取预算计数，只允许约定的一次真实请求，随后核对计数差值并冻结脱敏响应。发现额外请求立即停止发布检查。
