# 微信小程序上架文案与审核路径

## 基本资料

- 小程序名称：宜老天气通
- 建议版本号：`1.0.0`
- 品牌头像：`static/brand/yilao-avatar.png`，512 × 512 PNG
- 一句话简介：都昌县天气、官方预警和今日防护清单。
- 详细介绍：查看都昌县最近一次可用天气、七日预报、官方预警、社区脆弱性、避暑资源和热暴露网格，并把风险转换为补水、降温、联系家人等可执行行动。登录并主动同意隐私说明后，可管理家人档案、健康状态筛查、健康日记、用药记录和行动确认。相关功能只保存家庭自用记录，不提供诊断、处方、在线问诊或自动送达提醒。
- 搜索关键词：`都昌天气`、`天气预警`、`老人防暑`、`家庭照护`、`高温提醒`、`避暑资源`

个人主体提交时，以微信后台当前实际可选类目和资质提示为唯一准绳。首选与天气信息、生活工具、家庭记录实际功能一致且个人主体可选的类目。健康状态筛查、健康日记和用药记录必须在简介、隐私保护指引和审核备注中准确披露。若后台把这些真实功能归入个人主体不可选或需要机构资质的类目，立即停止提交，记录后台提示，再决定是否制作对所有用户一致生效的公开天气范围版本。禁止隐藏功能、仅对审核员关闭入口，或借用医疗诊断、互联网医院、在线问诊类目。

## 审核人员体验路径

1. 打开小程序，无需登录即可进入首页。
2. 依次打开“预报”“官方预警”“今日行动”“社区”“避暑资源”“热暴露 GIS”。
3. 在“我的”中查看隐私说明、数据来源、更新时间和医疗边界。
4. 如需审核登录功能，点击“微信登录”，主动勾选隐私同意后完成登录。
5. 登录后在“照护”中添加一位测试家人，完成天气行动评估、生活记录、日常用品服用备忘和今日行动。
6. 在“账号与隐私”中验证退出和申请注销入口。

审核备注建议写明：公共功能不要求审核员提供个人健康信息；测试健康记录可以使用虚构内容，提交后可以立即注销测试账号。

## 截图清单

开发者工具和真机验收时各留一套不含真实个人信息的截图：

1. 首页天气、风险与更新时间。
2. 七日预报或官方预警。
3. 社区与避暑资源。
4. 家庭照护首页，使用虚构称呼和无敏感内容的测试档案。
5. 账号、隐私与注销页，展示退出、注销和数据控制入口。

商店截图建议按以下顺序配文，使用同一台手机、同一字号和不含个人信息的审核数据：

1. `今天该注意什么，一眼看清`，首页天气、风险等级、更新时间和前三项行动同屏。
2. `官方预警与 7 天宜老天气`，展示预警来源、生效时间和逐日高低温。
3. `把提醒变成能完成的清单`，展示今日行动进度与复制给家人的入口。
4. `社区脆弱性与避暑资源`，展示社区背景指标、更新时间和可读资源列表。
5. `家庭记录由你掌控`，展示账号与隐私控制、账号删除入口和“只保存记录”的边界。登录前隐私同意另作审核路径截图，不与登录后账号页拼接成一张不真实图片。

截图中不得出现 AppSecret、天气 Key、Bearer Token、OpenID、真实老人姓名、慢病、日记、用药或服务器内部信息。
