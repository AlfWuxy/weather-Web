## 你只需要完成的操作

在微信公众平台的官方页面出现对应步骤时，按下面内容填写或操作。

| 页面字段或操作 | 直接填写或执行 |
| --- | --- |
| 主体类型 | `个人` |
| 职业信息（选填） | `无`，允许留空时也可以跳过 |
| 真实姓名、身份证件信息 | 按本人有效证件填写，只在微信官方页面输入 |
| 联系手机号、短信验证码 | 使用本人可正常接收验证码的手机号，只在微信官方页面输入 |
| 人脸核验或管理员确认 | 由本人按页面提示完成 |
| 联系邮箱 | 使用长期可访问的专用邮箱；由维护者现场读取官方页面，或由本人粘贴到本机无回显输入，禁止在聊天中发送邮箱全文 |
| 认证或其他费用 | 只支付微信官方页面当时明确展示的金额；付款前核对收款方、项目名称和订单状态 |
| 平台或开发者工具扫码确认 | 页面实际要求时，由本人使用管理员微信扫码或确认 |

完成某一步后，只需告诉维护者：`认证完成`、`短信完成`、`人脸完成`、`管理员核验完成` 或 `支付完成`。不要在聊天中发送身份证号、验证码、银行卡信息、AppSecret、私钥或未经遮盖的完整后台截图。

你无需编辑 `.env.wechat-release`，无需复制 AppID，也无需把 AppSecret 交给维护者。正式 AppID 已保存在本机私密配置；AppSecret 页面只需要你完成微信要求的管理员核验，后续录入由维护者在当前电脑本机完成。

个人主体通常不需要填写以下企业字段。页面若强制要求其中任何一项，先停止并截图非敏感区域：

- 营业执照
- 统一社会信用代码
- 企业法定代表人
- 对公银行账户
- 医疗机构执业许可
- 科普、公益或 IT 职业资质

## 我负责完成的操作

- 核对小程序名称、头像、简介、服务类目和审核文案。
- 根据实际代码填写隐私保护指引与用户协议，不扩大数据收集范围。
- 配置正式 AppID、AppSecret、HTTPS 合法域名和服务器环境。
- 服务器域名只配置 `request` 合法域名 `https://yilaoweather.org`；当前代码未使用 socket、uploadFile、downloadFile、UDP 或 TCP，这些字段保持空白。
- 保护密钥，不把 AppSecret、QWeather 凭据或上传私钥写入 Git、日志和小程序包。
- 完成后端迁移、30 分钟天气缓存策略、额度保护和一次受控正式天气烟测。
- 完成开发者工具编译、体验版、真机功能与大字号验收。
- 固定目标 commit、版本号、截图、回滚点并准备上传和审核材料；平台要求管理员确认时交由账号持有人操作。

## 需要你把页面交给我看的节点

以下页面只提供遮盖敏感信息后的截图，或把当前电脑交给维护者继续操作：

1. 账号首页，能看到主体类型和账号状态。
2. 小程序基本信息页，能看到名称审核状态。
3. 服务类目选择页，能看到平台实际提供的类目和资质要求。
4. 开发管理页，能看到 AppID 与 AppSecret 的“生成/重置”按钮状态。
5. 开发设置页，能看到服务器域名配置项。
6. 隐私保护指引页，能看到待填写的数据类型和审核状态。

AppSecret 生成后不要复制到聊天，也不要写入 `project.private.config.json`。可以把电脑留在该页面，由维护者直接录入本机权限为 `0600` 的私密发布表单；也可以由本人在维护者提供的本机隐藏输入提示中粘贴。本机隐藏输入不会回显，完成后应关闭或遮挡仍可能显示明文的官方页面。

为减少重复登录，正式文案与代码准备完成后再进行最后一次管理员会话。维护者会在同一会话内核对类目、保存遮盖敏感信息的证据、配置域名、录入 AppSecret，并准备上传页；账号持有人只处理页面临时要求的扫码、短信、人脸或付款。类目证据只有 24 小时有效，过期后需要重新截图核对。

## 必须暂停并核对的情况

遇到以下任一情况，先停止，不要自行猜选：

- 页面要求把个人主体改为企业、机构或媒体主体。
- 服务类目要求医疗、互联网新闻、公益募捐、专业科普等额外资质。
- 页面出现未讨论过的协议、授权范围、付费项目或自动续费。
- 维护者在场并已说明目的的首次生成或安全轮换 AppSecret，可以按既定发布流程继续；页面意外要求再次重置、删除、上传 AppSecret 或上传 QWeather 凭据时暂停核对。
- 页面显示的主体、名称、AppID 或域名与发布表单不一致。

## 最终交接口令

全部用户步骤完成后，发送：`个人主体步骤已完成，请继续发布核对`。

维护者会先复核私密发布表单、类目证据和工具链完整性。只有正式门禁全部通过后，才会部署、执行唯一一次真实天气烟测并上传审核版本。
