# 热暴露 GIS 发布与回滚清单

## 发布前

1. 从 `origin/main` 创建独立发布分支，避免捆绑其他未合并工作。
2. 运行 GIS、指标透明度、导航和全量回归测试。
3. 检查 GeoJSON 网格数、输入指纹、schema、去标识字段和本地 Leaflet 文件哈希。
4. 在生产环境创建仅管理员可读的代码与配置快照，记录发布前 Git SHA。

## 渐进发布

1. 合并经过检查的 Pull Request。
2. 从合并提交构建干净 release 工作树，只同步本次版本控制文件。
3. 生产环境保持 `FEATURE_HEAT_EXPOSURE_GIS=1`，重启应用。
4. 依次验证登录、桌面“更多”、移动抽屉、GIS 页面、GeoJSON、CSS、JavaScript 和 Leaflet 静态文件。
5. 经正式域名检查 Cloudflare 首次回源和后续缓存命中，并核对应用日志无新增异常。

## 快速回滚

1. 将 `FEATURE_HEAT_EXPOSURE_GIS=0`，重启应用。
2. 验证“更多”和移动入口消失，登录后的 GIS 路由返回 404，网站其他页面继续工作。

## 完整回滚

1. 恢复本次发布前代码与配置快照。
2. 重启应用并验证核心页面、定时任务和错误日志。
3. 若要求撤回公开 GeoJSON，同时移除本次新增静态产物并清理对应 Cloudflare 缓存 URL。
4. 保留回滚事件、前后 Git SHA、验证结果和执行时间。
